## 2026-06-28 - JSDOM Module Isolation Leak
**Learning:** In JSDOM testing environments, `jest.isolateModules` executes module instantiation in an isolated sandbox, but standard JSDOM elements (like `window.document.body` or mocked references) might not persist cleanly across those isolation boundaries. If the isolated module's IIFE or top-level execution triggers DOM setup (e.g. `new window.Sketch`), missing properties on stubs within that boundary (like `canvas.className` reading from undefined) will throw a hard-to-catch setup failure.
**Action:** When invoking `jest.isolateModules` to hit top-level branch setup execution blocks, explicitly suppress expected environment parsing errors by deeply mocking `window.console.error` *inside* the isolation block and immediately restoring it, or provide a deeply mocked sandbox execution context so the execution does not surface as a test framework crash in CI.
